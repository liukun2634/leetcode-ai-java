# [LeetCode 153. 寻找旋转排序数组中的最小值 (Find Minimum in Rotated Sorted Array)](https://leetcode.com/problems/find-minimum-in-rotated-sorted-array/)

> 难度：Medium　|　标签：数组、二分查找　|　**旋转数组对偶题 ⭐⭐⭐**

---

## 一、题目

已知一个长度为 `n` 的数组，预先按照升序排列，经由 `1` 到 `n` 次 **旋转** 后，得到输入数组。

请你找出并返回数组中的 **最小元素**。

要求 `O(log n)` 时间复杂度。

**约束**

- `n == nums.length`，`1 <= n <= 5000`
- 元素互不相同
- 数组保证旋转 1 到 n 次（包含 0 次的情况下结果仍为最小）

**示例**

| 输入 | 输出 |
|---|---|
| `[3,4,5,1,2]` | `1` |
| `[4,5,6,7,0,1,2]` | `0` |
| `[11,13]` | `11` |

---

## 二、解题思路（学习重点）

### 1. 关键观察：比较 `nums[mid]` 与 `nums[right]`

旋转数组分两段升序。最小值是"折点"。
- `nums[mid] > nums[right]`：折点（最小）在 **右半** → `l = mid + 1`
- `nums[mid] < nums[right]`：mid 已在右半（或未旋转），折点在 **左半（含 mid）** → `r = mid`

**为什么和 right 比而不是 left 比？**
- 与 `nums[right]` 比能 **明确区分左右段**：mid > right 一定 mid 在左段，最小在右；mid < right 一定 mid 在右段，最小在左（或就是 mid）。
- 与 `nums[left]` 比时，未旋转的情形会让两者总相等，需要特判。

> **学习点 ①**：旋转数组找最小值的标准模板 = **`nums[mid] vs nums[r]`**。这是 **二分模板"二段性"** 的体现。

### 2. 边界处理

- `r = mid`（而不是 `mid - 1`）：mid 可能就是最小值
- 循环条件 `l < r`：用半开区间，最终 l == r 即答案

### 3. 容易踩的坑

| 坑 | 处理 |
|---|---|
| 用 `nums[mid] vs nums[l]` 不能区分未旋转 | 用 `nums[mid] vs nums[r]` |
| `r = mid - 1` 漏掉 mid 是最小的情形 | `r = mid` |
| 循环 `l <= r` 配 `r = mid` 死循环 | 用 `l < r` |
| 元素含重复（LC 154）| 等于时 `r--`，最坏 O(n) |

---

## 三、详细解题步骤

**步骤 1**：初始化
```java
int l = 0, r = nums.length - 1;
```

**步骤 2**：二分循环 `while (l < r)`：
  1. `mid = (l + r) >>> 1;`
  2. 若 `nums[mid] > nums[r]` → 最小在右 → `l = mid + 1;`
  3. 否则（含相等 / 小于）→ 最小在左半含 mid → `r = mid;`

**步骤 3**：返回 `nums[l]`（此时 l == r）。

---

## 四、Java 题解

```java
class Solution {
    public int findMin(int[] nums) {
        int l = 0, r = nums.length - 1;
        while (l < r) {
            int mid = (l + r) >>> 1;
            if (nums[mid] > nums[r]) l = mid + 1;
            else                      r = mid;
        }
        return nums[l];
    }
}
```

**记忆口诀**：
> **"中比右大走右半，否则走左半含 mid。"**

## 六、示例验证

`nums = [4,5,6,7,0,1,2]`

| 轮 | l | r | mid | nums[mid] | nums[r] | 比较 | 收缩 |
|---|---|---|---|---|---|---|---|
| 1 | 0 | 6 | 3 | 7 | 2 | 7 > 2 | l = 4 |
| 2 | 4 | 6 | 5 | 1 | 2 | 1 < 2 | r = 5 |
| 3 | 4 | 5 | 4 | 0 | 1 | 0 < 1 | r = 4 |
| 退出 | 4 | 4 | — | — | — | — | `nums[4] = 0` ✅ |

## 七、复盘与延伸

### 一句话总结
> **比 nums[mid] 与 nums[r]：大走右半，否则走左半含 mid；l == r 即最小。**

### 新手常见疑问（FAQ）

**Q1：为什么必须和 right 比而不是 left？**
A：未旋转或单元素时，与 left 比会让 `nums[mid] >= nums[left]` 总成立，无法区分左右段。与 right 比能保持二段性。

**Q2：`r = mid` 而不是 `r = mid - 1` 为什么？**
A：mid 本身可能就是最小值（`nums[mid] < nums[r]` 时）。`r = mid - 1` 会跳过它。

**Q3：相等怎么办（LC 154）？**
A：本题元素唯一不会相等。LC 154 中 `nums[mid] == nums[r]` 时 `r--`（保守缩 1，最坏 O(n)）。

**Q4：未旋转（`[1,2,3]`）这套模板还对吗？**
A：对。整个数组都满足 `nums[mid] < nums[r]`，每次走左，最终 l = 0，返回 `nums[0] = 1`。

**Q5：找最大值怎么改？**
A：反向：`nums[mid] < nums[l]` 时最大在左半；否则在右半。或先找最小值，最大 = `nums[(idx + n - 1) % n]`。

### 面试官常见 follow-up
1. **"含重复元素呢（LC 154）？"** → 相等时 `r--`，最坏 O(n)。
2. **"搜索旋转数组中的某值（LC 33）？"** → 二分前先判哪半有序，再决定走哪边。
3. **"返回旋转点（最小值的下标）？"** → 直接返回 l（不是 nums[l]）。
4. **"如何检测数组到底旋转了几次？"** → 找到最小值后，旋转次数 = 最小值的下标。
5. **"未旋转 vs 已旋转 怎么区分？"** → `nums[0] <= nums[n-1]` → 未旋转。
6. **"分布式场景：数组按段切到多台机器？"** → 每台先找局部最小，再聚合时再二分。
