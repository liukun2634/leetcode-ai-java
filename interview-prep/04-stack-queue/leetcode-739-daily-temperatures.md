# LeetCode 739. 每日温度 (Daily Temperatures)

> 难度：Medium　|　标签：栈、单调栈、数组　|　**单调栈入门 ⭐⭐⭐**

---

## 一、题目

给定一个整数数组 `temperatures`，表示每天的温度，返回一个数组 `answer`，其中 `answer[i]` 是指对于第 `i` 天，下一个 **更高温度** 出现在几天后。如果气温在这之后都不会升高，请在该位置用 `0` 来代替。

**约束**

- `1 <= temperatures.length <= 10^5`
- `30 <= temperatures[i] <= 100`

**示例**

| 输入 | 输出 |
|---|---|
| `[73,74,75,71,69,72,76,73]` | `[1,1,4,2,1,1,0,0]` |
| `[30,40,50,60]` | `[1,1,1,0]` |
| `[30,60,90]` | `[1,1,0]` |

题目链接：<https://leetcode.cn/problems/daily-temperatures/>

---

## 二、解题思路（学习重点）

### 1. 单调栈是这类题的"自然语言"

题型："**对每个元素，找右边第一个更大/更小的元素**" → **单调栈**。
- **找右边第一个更大** → 单调 **递减** 栈
- **找右边第一个更小** → 单调 **递增** 栈

### 2. 单调递减栈模板

栈里存 **下标**（不是值）。从左到右遍历：

```text
for i = 0..n-1:
    while (!stack.isEmpty() && temps[stack.peek()] < temps[i]) {
        int j = stack.pop();
        ans[j] = i - j;
    }
    stack.push(i);
```

**核心思维**：每个下标入栈一次、出栈一次（最多）→ **总操作 O(n)**。

> **学习点 ①**：单调栈的栈里"一直没被更大元素打败的下标"，等待被打败时结算答案。同模板：LC 496/503（下一个更大）、LC 901（股票跨度）、LC 84（柱状图最大矩形，递增栈）。

### 3. 容易踩的坑

| 坑 | 处理 |
|---|---|
| 用 `Stack` 同步开销大 | 用 `ArrayDeque<Integer>` |
| 栈里存温度而非下标 | 算不出 `i - j` 天数差 |
| 写成"≤" 还是 "<"？ | "更高"用 `<`；若是"≥"也算更高则用 `<=` |
| 末尾栈里剩余没处理 | `ans` 数组默认 0，不需特别处理 |

---

## 三、详细解题步骤

**步骤 1**：开 ans 数组（默认全 0）和栈
```java
int n = temperatures.length;
int[] ans = new int[n];
Deque<Integer> stack = new ArrayDeque<>();
```

**步骤 2**：遍历每个下标 i：
  1. 当栈不空且 `temps[stack.peek()] < temps[i]`：
     - 弹出栈顶 `j`，记 `ans[j] = i - j`
  2. push i

**步骤 3**：返回 ans。

---

## 四、Java 题解

```java
class Solution {
    public int[] dailyTemperatures(int[] temperatures) {
        int n = temperatures.length;
        int[] ans = new int[n];
        Deque<Integer> stack = new ArrayDeque<>();

        for (int i = 0; i < n; i++) {
            while (!stack.isEmpty() && temperatures[stack.peek()] < temperatures[i]) {
                int j = stack.pop();
                ans[j] = i - j;
            }
            stack.push(i);
        }
        return ans;
    }
}
```

**记忆口诀**：
> **"递减栈存下标，遇到更高就结算，差值即等待天数。"**

## 五、复杂度

| 项 | 复杂度 |
|---|---|
| 时间 | **O(n)** —— 每个下标至多入/出栈各 1 次 |
| 空间 | O(n) 栈 |

## 六、示例验证

`temps = [73,74,75,71,69,72,76,73]`

| i | t | 操作 | stack (顶在右) | ans |
|---|---|---|---|---|
| 0 | 73 | push | [0] | [0,...] |
| 1 | 74 | 弹 0: ans[0]=1; push | [1] | [1,0,...] |
| 2 | 75 | 弹 1: ans[1]=1; push | [2] | [1,1,0,...] |
| 3 | 71 | push | [2,3] | — |
| 4 | 69 | push | [2,3,4] | — |
| 5 | 72 | 弹 4: ans[4]=1; 弹 3: ans[3]=2; push | [2,5] | [1,1,0,2,1,0,...] |
| 6 | 76 | 弹 5: ans[5]=1; 弹 2: ans[2]=4; push | [6] | [1,1,4,2,1,1,0,...] |
| 7 | 73 | push | [6,7] | — |

输出 `[1,1,4,2,1,1,0,0]` ✅（栈里剩 6,7 没结算，ans 默认 0）

## 七、复盘与延伸

### 一句话总结
> **单调递减栈：栈里存"还没被超越"的下标；新元素比栈顶大就结算。**

### 新手常见疑问（FAQ）

**Q1：为什么是 O(n) 而不是 O(n²)？**
A：每个下标最多入栈 1 次、出栈 1 次。while 内的总弹栈次数 ≤ n。总操作 ≤ 2n。

**Q2：栈里为什么存下标而非温度？**
A：算"几天后"需要下标差。存温度还得另存下标，麻烦。

**Q3：什么时候用递增栈？**
A：找"下一个**更小**元素"用递增栈（栈里严格递增，遇到更小就弹结算）。LC 84 是经典。

**Q4：循环数组怎么改（LC 503）？**
A：扫两遍数组（`for i = 0..2n-1`，访问 `temps[i % n]`），只在第一遍记录答案。

**Q5：从右往左遍历也可以吗？**
A：可以，单调递增栈，逻辑对称。常见写法是从左往右。

### 面试官常见 follow-up
1. **"下一个更大元素 I（LC 496）/ II（LC 503）？"** → 同模板，II 是循环数组扫两遍。
2. **"股票价格跨度（LC 901）？"** → 找左边第一个比它大的，单调栈往左找。
3. **"接雨水（LC 42）？"** → 单调递减栈算积水（横向按层）。
4. **"找下一个更小的元素？"** → 改成单调递增栈，比较换成 `>`。
5. **"如果允许相同温度也算更高？"** → 把 `<` 换成 `<=`。
6. **"二维版（每个格子向右下角找第一个更大）？"** → 单调栈失效，需要其他数据结构（线段树）。

### 同类型推荐（**单调栈家族**）
- LC 496. 下一个更大元素 I
- LC 503. 下一个更大元素 II（循环数组）
- LC 84. 柱状图中最大的矩形（递增栈）
- LC 42. 接雨水（递减栈算积水）
- LC 901. 股票价格跨度（往左找）
- LC 1019. 链表中的下一个更大节点
- LC 962. 最大宽度坡（变种）
