# LeetCode 22. 括号生成 (Generate Parentheses)

> 难度：Medium　|　标签：回溯、剪枝　|	|	**回溯剪枝模板 ⭐⭐⭐**

---

## 一、题目

数字 `n` 代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且 **有效的** 括号组合。

**约束**

- `1 <= n <= 8`

**示例**

| 输入 | 输出 |
|---|---|
| `n = 3` | `["((()))","(()())","(())()","()(())","()()()"]` |
| `n = 1` | `["()"]` |

---

## 二、解题思路（学习重点）

### 1. 暴力 → 剪枝的进化

**暴力**：枚举所有 `2n` 个位置上 `(` 或 `)` 的组合（`2^(2n)` 种），最后过滤出合法的。计算量大。

**剪枝**：在 **生成过程中** 就保证合法性 → 大幅减少分支。

### 2. 剪枝条件

设当前已经放了 `open` 个 `(` 和 `close` 个 `)`：

- **可以放 `(` 的条件**：`open < n`
- **可以放 `)` 的条件**：`close < open`（保证任意前缀 `(` ≥ `)`）

> **学习点 ①**：合法括号串的等价条件 = "**任意前缀左括号数 ≥ 右括号数，且总数相等**"。
> 把这个条件嵌入递归分支选择，就实现了 **生成即合法**，无需后过滤。

### 3. 容易踩的坑

| 坑 | 处理 |
|---|---|
| 用 String 拼接，每次新建对象 | 用 `StringBuilder`，回溯时 `deleteCharAt(len - 1)` |
| 终止条件写成 `open + close == n` | 应该是 `== 2 * n` |
| 剪枝条件写反（`open > close` 才能放 `)`） | 等价但需注意符号 |

---

## 三、详细解题步骤

**步骤 1**：初始化
```java
List<String> ans = new ArrayList<>();
StringBuilder sb = new StringBuilder();
```

**步骤 2**：定义递归 `dfs(open, close)`：
  1. **递归出口**：若 `sb.length() == 2 * n`，把 `sb.toString()` 加入 `ans`，return。
  2. **尝试放 `(`**：若 `open < n`：
     - `sb.append('(');`
     - `dfs(open + 1, close);`
     - `sb.deleteCharAt(sb.length() - 1);`
  3. **尝试放 `)`**：若 `close < open`：
     - `sb.append(')');`
     - `dfs(open, close + 1);`
     - `sb.deleteCharAt(sb.length() - 1);`

**步骤 3**：调用 `dfs(0, 0)`，返回 `ans`。

---

## 四、Java 题解

### 解法 A：回溯 + 剪枝（推荐）

```java
class Solution {
    public List<String> generateParenthesis(int n) {
        List<String> ans = new ArrayList<>();
        dfs(new StringBuilder(), 0, 0, n, ans);
        return ans;
    }
    private void dfs(StringBuilder sb, int open, int close, int n, List<String> ans) {
        if (sb.length() == 2 * n) {
            ans.add(sb.toString());
            return;
        }
        if (open < n) {
            sb.append('(');
            dfs(sb, open + 1, close, n, ans);
            sb.deleteCharAt(sb.length() - 1);
        }
        if (close < open) {
            sb.append(')');
            dfs(sb, open, close + 1, n, ans);
            sb.deleteCharAt(sb.length() - 1);
        }
    }
}
```

**记忆口诀**：
> **"左未满放左，右少于左放右；满 2n 收一份。"**

### 解法 B：动态规划（构造法）

`dp[i]` = `i` 对括号能生成的所有有效组合。递推：
$$dp[i] = \bigcup_{j=0}^{i-1} \{\;"("+a+")"+b\;\mid\; a \in dp[j],\, b \in dp[i-1-j]\;\}$$

## 六、示例验证

`n = 3` 的递归树（精简）：

```
""
├ ( [open=1]
│   ├ (( [open=2]
│   │   ├ ((( [open=3] → 必须放)
│   │   │   └ ((() → ((()) → ((())) ✅
│   │   └ (() → 放(or)
│   │       ├ (()( → (()() → (()()) ✅
│   │       └ (()) → (())( → (())() ✅
│   └ () → 放(
│       ├ ()( → ()(( → ()(() → ()(()) ✅
│       └ ()( → 放) → ()() → ()()( → ()()() ✅
```

5 个组合 ✅（对应 Catalan(3) = 5）

## 七、复盘与延伸

### 一句话总结
> **"放左" 与 "放右" 各加一个剪枝条件，生成即合法。**

### 新手常见疑问（FAQ）

**Q1：为什么是 `close < open` 而不是 `close < n`？**
A：合法括号串的必要条件是“任意前缀中左括号个数 ≥ 右括号个数”。`close < open` 保证加右括号后仍有 close ≤ open；`close < n` 只限制总量，会生成 `())` 这种非法串。

**Q2：为什么用 `StringBuilder` 而不是 `String`？**
A：String 不可变，每次 `+` 都新建对象，额外复杂度与内存；StringBuilder 可在原处附加与 `deleteCharAt` 撤销。

**Q3：为什么 `4^n / √n`？**
A：合法括号串个数 = Catalan(n) = `C(2n,n)/(n+1)`，渐近为 `4^n / √n·n`。面试不需推导，能说出“Catalan 数”名字加分。

**Q4：合法括号串能不能用动态规划生成？**
A：能：dp[i] = 所有 `(a)b` 的并集，其中 a 是 dp[j]、b 是 dp[i-1-j]，j=0..i-1。代码不作为主解，面试可作亮点提及。

**Q5：如果要多种括号（`()` `[]` `{}`）？**
A：递归中多加“需要闭合什么括号”的栈，能放左括号时压入对应右括号，放右括号时必须与栈顶匹配。

### 面试官常见 follow-up
1. **"判断给定字符串是否合法呢？"** → 计数 / 栈打包判断。即 **LC 20**。
2. **"最长有效括号子串呢？"** → 栈存下标或 DP。即 **LC 32**。
3. **"删除最少括号使之合法（返回所有方案）？"** → BFS：按层扩展删除，一旦出现合法字符串就不再增加层。即 **LC 301**。
4. **"生成不同括号类型的所有合法串？"** → 栈助助的回溯；需为每种类型单独计数。
5. **"有多少个不同的合法串（只求数量）？"** → 直接输出 Catalan(n)；DP `f[i] = sum(f[j]*f[i-1-j])`。
6. **"打印流场景，一个个输出不能一次性存？"** → 回溯生成过程中直接 print 即可，不使用 `ans` 列表。

### 同类型推荐（**回溯剪枝家族**）
- LC 20. 有效的括号
- LC 32. 最长有效括号
- LC 301. 删除无效的括号
- LC 1249. 移除无效的括号
- LC 921. 使括号有效的最少添加
- LC 17. 电话号码字母组合（无剪枝的纯回溯）
- LC 39. 组合总和（剪枝：sum 超过 target 提前返回）
