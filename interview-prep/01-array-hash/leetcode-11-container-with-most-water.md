# LeetCode 11. 盛最多水的容器 (Container With Most Water)

> 难度：Medium　|　标签：数组、双指针、贪心　|　**双指针入门 ⭐⭐⭐**

---

## 一、题目

给定一个长度为 `n` 的整数数组 `height`。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])`。

找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。返回容器可以储存的最大水量。

**约束**

- `n == height.length`，`2 <= n <= 10^5`
- `0 <= height[i] <= 10^4`

**示例**

| 输入 | 输出 |
|---|---|
| `[1,8,6,2,5,4,8,3,7]` | `49`（i=1,j=8，宽 7 × 高 min(8,7)=7） |
| `[1,1]` | `1` |

题目链接：<https://leetcode.cn/problems/container-with-most-water/>

## 二、解题思路（学习重点）

### 1. 与 LC 42 接雨水的区别

- **LC 42**：所有柱子都参与，每个位置都可能存水。
- **LC 11**：**只选两条线**，组成一个矩形容器；中间的柱子不影响（假装是水）。

容器面积 = `min(height[l], height[r]) × (r - l)`，要在所有 `(l, r)` 对中求最大。

### 2. 暴力 O(n²) → 双指针 O(n) 的关键洞察

**暴力**：枚举所有 (l, r) 对 O(n²)。

**双指针**：`l = 0, r = n-1` 从两端往中间收缩。每次：
- 计算当前面积，更新最大值
- **移动较矮的一端**（往中间走）

**为什么移动较矮的一端是对的**：当前 `min(h[l], h[r])` 由较矮一侧决定。如果移动较高的一侧，宽度变小、高度上限不变（仍受较矮端约束），面积只会变小或不变。**只有移动较矮的一侧才可能让 min 变大**，从而面积有机会变大。

> **学习点 ①**：双指针的"该移动哪一边"，关键是分析"移动后哪一边的状态有改善空间"。

### 3. 容易踩的坑

| 坑 | 处理 |
|---|---|
| 移动较高的一端 → 漏最优解 | 必须移动 **较矮** 的一端 |
| 同时移动两端 → 可能漏解 | 每轮只移动一端 |
| 高度相等时 | 移哪边都行，结果一样 |

---

## 三、详细解题步骤

**步骤 1**：初始化
```java
int l = 0, r = height.length - 1, max = 0;
```

**步骤 2**：循环 `while (l < r)`：
  1. 算面积 `area = Math.min(height[l], height[r]) * (r - l);`
  2. 更新 `max = Math.max(max, area);`
  3. 移动较矮一端：
     - 若 `height[l] < height[r]` → `l++`
     - 否则 → `r--`

**步骤 3**：返回 `max`。

---

## 四、Java 题解

```java
class Solution {
    public int maxArea(int[] height) {
        int l = 0, r = height.length - 1, max = 0;
        while (l < r) {
            int area = Math.min(height[l], height[r]) * (r - l);
            max = Math.max(max, area);
            if (height[l] < height[r]) l++;
            else r--;
        }
        return max;
    }
}
```

**记忆口诀**：
> **"两端开始，算面积，移较矮，更新 max。"**

## 六、示例验证

`height = [1,8,6,2,5,4,8,3,7]`

| 步 | l | r | h[l] | h[r] | area | max | 移动 |
|---|---|---|---|---|---|---|---|
| 1 | 0 | 8 | 1 | 7 | min(1,7)*8=8 | 8 | l++ |
| 2 | 1 | 8 | 8 | 7 | min(8,7)*7=**49** | **49** | r-- |
| 3 | 1 | 7 | 8 | 3 | 3*6=18 | 49 | r-- |
| 4 | 1 | 6 | 8 | 8 | 8*5=40 | 49 | r-- |
| 5 | 1 | 5 | 8 | 4 | 4*4=16 | 49 | r-- |
| ... | ... | ... | ... | ... | ... | 49 | ... |

输出 `49` ✅

## 七、复盘与延伸

### 一句话总结
> **从两端往中间收缩，每次算面积并移动较矮端；较矮端有改善空间，较高端动了只会让面积变小。**

### 新手常见疑问（FAQ）

**Q1：为什么移动较高的一端不行？**
A：移动较高端后，宽度变小，min 仍由较矮端决定（不会变大），所以面积一定不变或更小。漏掉的最优组合不可能存在。

**Q2：高度相等时移哪边？**
A：任选一边都对。两边可移性等价，结果不变。

**Q3：和 LC 42 接雨水的双指针思路有什么共同点？**
A：都是"两端往中间逼近，每次处理较矮一端"。区别在于 LC 42 维护两侧 max 用于结算水量，LC 11 直接用当前 min 算容器。

**Q4：贪心的正确性如何证明？**
A：反证。设最优解是 `(i, j)`，且 `i, j` 不是双指针的某一步。若 l 走到 i 之前停下，意味着某一步 l 超过 i，但 l 只在 `h[l] < h[r]` 时移动，超过 i 时 `h[l-1] < h[r-某]`，矛盾推导可证最优解必然被覆盖。

**Q5：会不会超时？**
A：O(n) 时间，n=10⁵ 时只需 10⁵ 步，远不会 TLE。

### 面试官常见 follow-up
1. **"接雨水（LC 42）的解法和这题差异？"** → LC 42 算每个位置水量，需要前后缀 max；LC 11 只算两端容器，移较矮端即可。
2. **"如果柱子可以旋转，求三维容器？"** → 类似多维双指针，复杂度上升；面试少见。
3. **"如果允许选 k 条线组成多边形？"** → 大幅升级，需要凸包 + 几何。
4. **"如果加上 cost 限制（选某条线有代价）？"** → 转化为带约束优化，可能 DP。
5. **"流式数据，每来一条新线就报告最大容器？"** → 难，需维护「左边最高 / 右边最高」类似单调栈结构。
6. **"如何返回最大容器的左右下标？"** → 更新 max 时同步记录 `bestL = l, bestR = r`。
