# [LeetCode 56. 合并区间 (Merge Intervals)](https://leetcode.com/problems/merge-intervals/)

> 难度：Medium　|　标签：数组、排序、贪心　|　**区间问题模板 ⭐⭐⭐**

---

## 一、题目

以数组 `intervals` 表示若干个区间的集合，其中 `intervals[i] = [start_i, end_i]`。请你 **合并所有重叠的区间**，并返回一个 **不重叠的区间数组**，该数组需恰好覆盖输入中的所有区间。

**约束**

- `1 <= intervals.length <= 10^4`
- `0 <= start <= end <= 10^4`

**示例**

| 输入 | 输出 |
|---|---|
| `[[1,3],[2,6],[8,10],[15,18]]` | `[[1,6],[8,10],[15,18]]` |
| `[[1,4],[4,5]]` | `[[1,5]]`（端点接触也算重叠） |

---

## 二、解题思路（学习重点）

### 1. 区间题第一步：**按起点排序**

排序之后，任意一个区间只可能与 **它后面** 的区间产生重叠（前面的已经处理过）。这把无序的"两两比较"O(n²) 降到 O(n log n) + O(n) 的扫描。

**为什么起点排序后“只看上一个”就够**：起点排序后，如果当前区间与“已合并列表的最后一个”不重叠，它必定与列表里更早的区间也不重叠（更早的 end ≤ 最后一个的 end < 当前的 start）。不需要回头。

> **学习点 ①**：**绝大多数区间题，先按 start 排序**。例外是按 end 排序的 LC 435（无重叠区间）/ LC 452（用最少箭引爆气球）。

### 2. 一次扫描合并

维护一个结果列表 `merged`，遍历每个区间 `[s, e]`：
- 若 `merged` 为空，或 `s > merged.last.end`（**没碰上**）→ 直接加入
- 否则（**碰上了**）→ 把 `merged.last.end` 更新为 `max(merged.last.end, e)`

判断"碰上"的关键：**当前区间起点 `s` ≤ 上一个区间终点 `e_prev`**。注意题目要求 `[1,4],[4,5]` 也算重叠 → 用 `<=`。

### 3. 容易踩的坑

| 坑 | 处理 |
|---|---|
| 不排序，直接两两比较 | O(n²)，会 TLE |
| 合并时只更新 end，没考虑前区间已经包含后区间（`[1,10],[2,3]`） | `max(end_prev, e)` 而不是直接 `=e` |
| 用 `<` 漏掉端点接触情况 | 必须 `<=` |

---

## 三、详细解题步骤

**步骤 1**：边界处理
- 若 `intervals.length <= 1`，直接返回原数组。

**步骤 2**：按起点升序排序
```java
Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
```
（极端大值可能溢出时用 `Integer.compare(a[0], b[0])`）

**步骤 3**：创建结果列表
```java
List<int[]> merged = new ArrayList<>();
```

**步骤 4**：把第一个区间放入 `merged`：`merged.add(intervals[0]);`

**步骤 5**：从 `i = 1` 开始遍历：
  1. 取 `cur = intervals[i]`；
  2. 取 `last = merged.get(merged.size() - 1);`（**注意取的是数组引用，原地修改**）
  3. 如果 `cur[0] <= last[1]` （重叠）：
     - 更新 `last[1] = Math.max(last[1], cur[1]);`
  4. 否则（不重叠）：
     - `merged.add(cur);`

**步骤 6**：返回 `merged.toArray(new int[0][]);`

---

## 四、Java 题解

```java
class Solution {
    public int[][] merge(int[][] intervals) {
        if (intervals.length <= 1) return intervals;
        Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

        List<int[]> merged = new ArrayList<>();
        merged.add(intervals[0]);
        for (int i = 1; i < intervals.length; i++) {
            int[] last = merged.get(merged.size() - 1);
            int[] cur  = intervals[i];
            if (cur[0] <= last[1]) {                    // 重叠
                last[1] = Math.max(last[1], cur[1]);    // 直接原地扩张
            } else {
                merged.add(cur);
            }
        }
        return merged.toArray(new int[0][]);
    }
}
```

**记忆口诀**：
> **"先排序、扫一遍、能合就合、不能就开新区间。"**

## 六、示例验证

`[[1,3],[2,6],[8,10],[15,18]]` → 排序后顺序不变。

| i | cur | last | 重叠？ | merged |
|---|---|---|---|---|
| init | — | — | — | `[[1,3]]` |
| 1 | [2,6] | [1,3] | `2 ≤ 3` ✅ | `[[1,6]]`（last[1]=max(3,6)） |
| 2 | [8,10] | [1,6] | `8 ≤ 6` ❌ | `[[1,6],[8,10]]` |
| 3 | [15,18] | [8,10] | `15 ≤ 10` ❌ | `[[1,6],[8,10],[15,18]]` |

## 七、复盘与延伸

### 一句话总结
> **区间问题先排序；排序后区间合并 = 一次线性扫描判重叠。**

### 新手常见疑问（FAQ）

**Q1：为什么是 `cur[0] <= last[1]` 而不是 `<`？**
A：看题目对“端点接触”的定义。本题 `[1,4]` 和 `[4,5]` 算重叠（要合并为 `[1,5]`），用 `<=`。若题目明确“端点接触不算”，才用 `<`。

**Q2：`Arrays.sort(intervals, (a,b) -> a[0] - b[0])` 会不会溢出？**
A：当 `a[0]` 很大 / `b[0]` 很小时（如 `Integer.MIN_VALUE`）减法可能溢出，产生错误比较。安全写法：`Integer.compare(a[0], b[0])`。本题范围 `[0, 10^4]` 不溢出，但面试能说出这点是加分项。

**Q3：能不能原地修改 intervals，不开 `merged` 列表？**
A：可以。排序后用指针 `j`（结果末尾）在原数组写入，比较与覆盖 `intervals[i]` vs `intervals[j]`。最后 `Arrays.copyOf(intervals, j+1)`。

**Q4：合并时能直接写 `last[1] = cur[1]` 吗？**
A：不能。反例 `[[1,10],[2,3]]`：`cur=[2,3]` 重叠，但 `last[1]=10 > 3`。必须 `max(last[1], cur[1])` 保证不缩。

**Q5：输入为空怎么处理？**
A：本题保证 `length >= 1`，但面试中口述时加一句“边界：空数组直接返回”是加分项。

### 面试官常见 follow-up
1. **"插入一个新区间到已合并的区间列表？"** → 二分找到插入位置 + 合并；或打散重走。即 **LC 57**。
2. **"最少需要多少个会议室？"** → 按 start 排序 + 最小堆跟踪 end；或开始/结束事件差分扫描。即 **LC 253**。
3. **"在最多可参加会议数量上取最大（无重叠区间数量）？"** → 按 **end** 排序 贪心。即 **LC 435 / 452**。
4. **"两个已排序的区间列表求交集？"** → 双指针各走一边，取 `max(start)` 到 `min(end)`，较小 end 者前进。即 **LC 986**。
5. **"区间被动态添加、需随时查询总覆盖长度？"** → `TreeMap` 维护不重叠区间集，插入时合并附近区间。即 **LC 715. Range Module**。
6. **"区间数量极大，内存不够？"** → 外部排序（归并） + 流式扫描合并。

### 自我提问
1. 为何按起点排序就够？→ 起点排序后，与"上一个 merged 区间"的关系完全决定了是否合并；不会出现需要"回头"的情况。
2. `cur[0] <= last[1]` 还是 `<`？→ 看题目对端点接触的定义。本题接触算重叠，用 `<=`。
3. 如何修改原地（不开新列表）？→ 用双指针 `j` 写入 `intervals`，最后 `Arrays.copyOf(intervals, j+1)`。
4. 加入一个新区间到已合并列表（LC 57）？→ 二分找到插入位置后再走合并；或线性扫一遍。
