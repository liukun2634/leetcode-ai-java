# LeetCode 238. 除自身以外数组的乘积 (Product of Array Except Self)

> 难度：Medium　|　标签：数组、前缀和、前后缀分解　|	|	**前后缀技巧 ⭐⭐⭐**

---

## 一、题目

给你一个整数数组 `nums`，返回 数组 `answer`，其中 `answer[i]` 等于 `nums` 中除 `nums[i]` 之外其余各元素的 **乘积**。

题目数据 **保证** 数组 `nums` 中任意元素的全部前缀元素和后缀的乘积都在 32 位整数范围内。

**进阶**：请 **不要使用除法**，且在 `O(n)` 时间复杂度内完成此题。**输出数组不计入空间复杂度**，要求 `O(1)` 额外空间。

**约束**

- `2 <= nums.length <= 10^5`

**示例**

| 输入 | 输出 |
|---|---|
| `[1,2,3,4]` | `[24,12,8,6]` |
| `[-1,1,0,-3,3]` | `[0,0,9,0,0]` |

题目链接：<https://leetcode.cn/problems/product-of-array-except-self/>

## 二、解题思路（学习重点）

### 1. 为什么不能用除法？

- 题目明确不许除法（面试也常加这个限制）
- 含 0 时除法直接出问题

### 2. 关键思路：**前缀积 × 后缀积**

**从最期待的“总积 ÷ 自己”走不通后怎么想**：
- 除法被禁，且遇 0 会出问题。重新拆解：`ans[i]` = 「 i 左侧所有元素乘积 」×「 i 右侧所有元素乘积 」。
- 左侧乘积可以从左到右一遍累乘得到（前缀积）；右侧乘积从右到左一遍累乘得到（后缀积）。两者相乘即是答案。

对每个位置 `i`，"除自己以外的乘积" =
$$\text{answer}[i] = (\text{nums}[0] \cdot ... \cdot \text{nums}[i-1]) \times (\text{nums}[i+1] \cdot ... \cdot \text{nums}[n-1])$$

设：
- `L[i]` = `i` 左侧所有元素的乘积（`L[0] = 1`）
- `R[i]` = `i` 右侧所有元素的乘积（`R[n-1] = 1`）

则 `answer[i] = L[i] * R[i]`。

**两次扫描**即可：左到右算 `L`，右到左算 `R`。

> **学习点 ①**：**"除自身"** 类问题的通用模板 = **前缀 + 后缀**。同样适用于"最大乘积 / 最大和 / 最大值"等。

### 3. O(1) 额外空间优化

把 `answer` 数组先存 `L`，然后从右向左扫一遍用一个变量 `R` 滚动相乘即可。

### 4. 容易踩的坑

| 坑 | 处理 |
|---|---|
| 用除法："总积 / nums[i]" | 含 0 会爆；题目禁止 |
| `L[0]` 初始化为 0 | 应为 1（空乘积是 1） |
| 二次扫描时把 `answer[i] *= R` 写错位置 | 必须**先乘 R 再更新 R** |

## 三、详细解题步骤（O(1) 空间版）

**步骤 1**：开 `int[] ans = new int[n];`

**步骤 2**：左到右扫，让 `ans[i] = i 左侧所有元素的乘积`
  ```text
  ans[0] = 1
  for i = 1..n-1:
      ans[i] = ans[i-1] * nums[i-1]
  ```
  扫完后 `ans = [1, nums[0], nums[0]*nums[1], ...]`

**步骤 3**：右到左扫，用变量 `R` 累乘右侧
  ```text
  R = 1
  for i = n-1..0:
      ans[i] = ans[i] * R     // 此时 ans[i] 是左积，乘上右积
      R = R * nums[i]         // 然后把当前元素并入 R
  ```

**步骤 4**：返回 `ans`。

> 注意 **顺序**：`ans[i] *= R` 必须在 `R *= nums[i]` 之前！否则 `R` 会包含 `nums[i]` 自己。

## 四、Java 题解

```java
class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] ans = new int[n];

        // Pass 1：左侧前缀积
        ans[0] = 1;
        for (int i = 1; i < n; i++) {
            ans[i] = ans[i - 1] * nums[i - 1];
        }

        // Pass 2：右侧后缀积，用一个变量滚动
        int R = 1;
        for (int i = n - 1; i >= 0; i--) {
            ans[i] *= R;        // 先乘
            R *= nums[i];       // 再更新
        }
        return ans;
    }
}
```

**记忆口诀**：
> **"左扫前缀积，右扫滚后缀，乘完再滚动。"**

## 六、示例验证

`nums = [1, 2, 3, 4]`

**Pass 1**（左前缀积）：
| i | 0 | 1 | 2 | 3 |
|---|---|---|---|---|
| ans | 1 | 1 | 2 | 6 |

**Pass 2**（右滚动）：
| i | ans[i] 乘前 | R 乘前 | ans[i] *= R | R *= nums[i] |
|---|---|---|---|---|
| 3 | 6 | 1 | 6 * 1 = 6 | 1 * 4 = 4 |
| 2 | 2 | 4 | 2 * 4 = 8 | 4 * 3 = 12 |
| 1 | 1 | 12 | 1 * 12 = 12 | 12 * 2 = 24 |
| 0 | 1 | 24 | 1 * 24 = 24 | 24 * 1 = 24 |

输出 `[24, 12, 8, 6]` ✅

## 七、复盘与延伸

### 一句话总结
> **前缀积 × 后缀积 = 除自身以外的乘积；用输出数组装左积，再用一个变量滚动右积达成 O(1) 空间。**

### 新手常见疑问（FAQ）

**Q1：为什么不可以 `total / nums[i]`？**
A：三个原因：① 题目明禁；② 含 0 时分母为 0；③ 面试场景下除法可能有精度问题（浮点）或越界问题。面试主动领会“不能依赖除法”是加分项。

**Q2：含一个 0 / 两个 0 分别会怎样？**
A：算法天然正确：一个 0→只有那个 0 位置非 0（左右除了 0 外的乘积）；两个或以上 0→全为 0。

**Q3：为什么 `ans[i] *= R` 必须在 `R *= nums[i]` 之前？**
A：如果先 `R *= nums[i]`，`R` 就包含了 `nums[i]` 自己，违反“除自身”。顺序错误是静默错，难发现。

**Q4：输出数组不计空间是什么意思？**
A：这是 LeetCode 常见说法——“输出是必需的，不算额外空间”，考查有没有 **额外** 开函数所需的 O(1) 临时变量。面试可以反问“输出空间是否计在复杂度内”清晰需求。

**Q5：乘积会不会溢出？**
A：题目保证所有前后缀乘积在 32 位范围内，`int` 够用。若面试护送去掉这个保证，要改 `long` 或评估需要 BigInteger。

### 面试官常见 follow-up
1. **"允许用除法但要处理 0，怎么写？"** → 统计 0 的个数：≠ 一个都是总积÷自身；= 1 个 只有那个 0 位非零（乘非零部分）；≥ 2 个 全 0。
2. **"求除自身以外的最大乘积、最小乘积呢？"** → 同时维护前后缀的 max 与 min（负×负转正）。
3. **"求除自身以外的和？"** → 类似：`ans[i] = totalSum - nums[i]`，一行解。
4. **"二维版：求除本行本列以外的矩阵元素和/乘积？"** → 前缀和/积推广到二维，或“总和 − 本行总 − 本列总 + 本元素”。
5. **"数组会动态更新某个位置，之后查询 productExceptSelf？"** → 维护 0 个数 + 非零乘积；查询 O(1)，更新 O(1)。
6. **"极大数据下溢出如何处理？"** → `long`/`BigInteger`；或取模下计算（需保证模数是质数以使逆元可用）。

### 自我提问
1. 含 0 怎么办？→ 算法天然正确：若数组含一个 0，结果中只有那个位置非 0；含两个或以上 0，全为 0。
2. 为什么不能用 `total / nums[i]`？→ 含 0 时分母为 0；且题目不允许除法。
3. 推广到 **最大/最小乘积**？→ 同时维护前后缀的 max/min。
4. 推广到二维 / 三维？→ 类比：累乘前缀矩阵。
