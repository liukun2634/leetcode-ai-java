# LeetCode 33. 搜索旋转排序数组 (Search in Rotated Sorted Array)

> 难度：Medium　|　标签：数组、二分查找　|　**二分变形必刷 ⭐⭐⭐**

---

## 一、题目

整数数组 `nums` 原本按 **升序排列**，且各元素 **互不相同**。在传给函数之前，`nums` 在某个未知下标 `k` 上进行了 **旋转**，使数组变为 `[nums[k], nums[k+1], ..., nums[n-1], nums[0], ..., nums[k-1]]`。

给你旋转后的数组和整数 `target`，找出 `target` 在数组中的下标；如果不存在，返回 `-1`。

**要求 O(log n) 时间复杂度**。

**约束**

- `1 <= nums.length <= 5000`，元素互不相同

**示例**

| 输入 | 输出 |
|---|---|
| `nums = [4,5,6,7,0,1,2], target = 0` | `4` |
| `nums = [4,5,6,7,0,1,2], target = 3` | `-1` |
| `nums = [1], target = 0` | `-1` |

题目链接：<https://leetcode.cn/problems/search-in-rotated-sorted-array/>

---

## 二、解题思路（学习重点）

### 1. 关键观察：旋转后数组"半段仍然有序"

设中点为 `mid`，对比 `nums[mid]` 与 `nums[left]`（或与 `nums[right]`）：

- 若 `nums[left] <= nums[mid]`：**左半 `[left, mid]` 严格有序**
- 否则：**右半 `[mid, right]` 严格有序**

> **学习点 ①**：旋转后的数组里至少有一半区间仍然单调有序，二分就在 **"有序的那半"** 里判断 target 是否落入。

### 2. 二分骨架

每一轮：
1. 算 `mid`。
2. 如果 `nums[mid] == target`，返回。
3. 找出哪一半有序。
4. 判断 `target` 是否落在那一段的 `[left, mid-1]` 或 `[mid+1, right]`；
   - 落在 → 收到该侧
   - 不落在 → 收到另一侧

### 3. 选模板的细节：用 `nums[left] <= nums[mid]` 还是 `nums[mid] <= nums[right]`？

两种都可以，但 **`<=` 是必要的**（处理 `left == mid` 的边界，如长度为 2 的窗口）。本题元素互不相同，无须额外处理重复。

> 如果题目允许 **重复元素**（LC 81），还需在 `nums[left] == nums[mid] == nums[right]` 时 `left++; right--;`，**复杂度退化为最坏 O(n)**。

### 4. 容易踩的坑

| 坑 | 处理 |
|---|---|
| 左右边界用 `<` 还是 `<=`？ | 闭区间 `[l, r]` 用 `while (l <= r)` |
| `mid` 溢出 | 用 `l + (r - l) / 2` 或 `(l + r) >>> 1` |
| 判断有序段时漏掉 `<=` | 如 `[3, 1]`，`l=0, mid=0`，`nums[l]==nums[mid]` 需归到"左半有序" |
| target 在有序段 **端点** | 边界用 `nums[l] <= target < nums[mid]`（含 l，不含 mid） |

## 三、Java 题解（推荐）

```java
class Solution {
    public int search(int[] nums, int target) {
        int l = 0, r = nums.length - 1;
        while (l <= r) {
            int mid = (l + r) >>> 1;
            if (nums[mid] == target) return mid;

            if (nums[l] <= nums[mid]) {                  // 左半有序
                if (nums[l] <= target && target < nums[mid]) r = mid - 1;
                else l = mid + 1;
            } else {                                     // 右半有序
                if (nums[mid] < target && target <= nums[r]) l = mid + 1;
                else r = mid - 1;
            }
        }
        return -1;
    }
}
```

**记忆口诀**：
> **"先看哪半有序，再看 target 是否落在那段。"**

## 五、示例验证

`nums = [4,5,6,7,0,1,2], target = 0`

| 轮 | l | r | mid | nums[mid] | 判断 | 收缩 |
|---|---|---|---|---|---|---|
| 1 | 0 | 6 | 3 | 7 | 左半 [4,5,6,7] 有序；0 不在 [4,7) | l = 4 |
| 2 | 4 | 6 | 5 | 1 | 左半 [0,1] 有序；0 ∈ [0,1) | r = 4 |
| 3 | 4 | 4 | 4 | 0 | **命中** ✅ | 返回 4 |

## 六、复盘与延伸

### 一句话总结
> **旋转数组二分：每次先判出"哪半是有序的"，再用普通有序段比较把 target 引到那一半。**

### 新手常见疑问（FAQ）

**Q1：为什么不能直接按标准二分比较 `nums[mid]` 与 `target`？**
A：旋转后 `nums[mid]` 右侧可能有比 `target` 小的环绕值（如 `[4,5,6,7,0,1,2]`，mid=3 是 7，target=0 要去右边，但右边值比 7 小）。要先定位哪半有序才能正确可靠比较。

**Q2：为什么 `nums[l] <= nums[mid]` 是 `<=` 而不是 `<`？**
A：处理 `l == mid` 的边界（如长度为 2 的窗口 `[3,1]`）。`l == mid` 时 `nums[l]==nums[mid]`，必须归为"左半有序"（单点总是有序的）。

**Q3：判 target 在哪半为什么是 `[nums[l], nums[mid])`（含 l 不含 mid）？**
A：`nums[mid] == target` 已在上一步返回，所以不考虑。含 `l` 是为了照顾 target = nums[l] 的情况。

**Q4：如果数组未旋转（k=0）这代码还对吗？**
A：对。整个数组是"左半有序"进入标准二分，表现与反转后一致。

**Q5：含重复元素呢（LC 81）？**
A：额外处理 `nums[l] == nums[mid] == nums[r]` 的退化：`l++; r--;` 各退一步。最坏复杂度 O(n)（全相同值时）。

### 面试官常见 follow-up
1. **"求旋转数组的最小值下标？"** → 二分比较 `nums[mid]` 与 `nums[r]`：mid > r 走右，否则走左。即 **LC 153**。
2. **"含重复元素的旋转数组查找/最小？"** → LC 81 / 154，额外处理三者相等退化。
3. **"未知些许但能 O(log n) 的其他“似乱实序”场景？"** → LC 162 寻找峰值：无序数组二分比邻，选递增一侧。
4. **"仅查找是否存在，不要下标？"** → 代码一样，返回 `true/false` 即可。
5. **"二分 “答案范围” 怎么理解（如 LC 875）？"** → 不是在数组上二分，是在「可能答案区间 [lo, hi]」上二分，需设计 `check(mid)` 函数。
6. **"两个有序数组的中位数（LC 4）为什么难？"** → 需在两数组上同时二分，划分点互相限制，边界繁琐。
