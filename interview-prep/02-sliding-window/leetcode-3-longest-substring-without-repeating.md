# [LeetCode 3. 无重复字符的最长子串 (Longest Substring Without Repeating Characters)](https://leetcode.com/problems/longest-substring-without-repeating-characters/)

> 难度：Medium　|　标签：哈希表、字符串、滑动窗口　|　**滑动窗口模板题 ⭐⭐⭐**

---

## 一、题目

给定一个字符串 `s`，请你找出其中不含有 **重复字符** 的 **最长子串** 的长度。

**约束**

- `0 <= s.length <= 5 * 10^4`
- `s` 由英文字母、数字、符号和空格组成

**示例**

| 输入 | 输出 | 说明 |
|---|---|---|
| `"abcabcbb"` | `3`  | `"abc"` |
| `"bbbbb"` | `1` | `"b"` |
| `"pwwkew"` | `3` | `"wke"`（不是 "pwke"，子串要连续） |

---

## 二、解题思路（学习重点）

### 1. 暴力 → 滑动窗口的进化

- **暴力**：枚举所有 `(i, j)` 区间检查无重复，O(n³)。
- **关键观察**：固定左端 `l`，当右端 `r` 扩到 **第一次重复** 时，再继续扩 `r` 的结果只可能更差；
  此时应当 **把 `l` 缩到重复字符之后**，继续扩 `r`。
  → 这就是 **滑动窗口（双指针）**：`l`、`r` 都只向右移动，**单调** 不回头。

### 2. 滑动窗口万能骨架

```text
l = 0
for r = 0..n-1:
    把 s[r] 加入窗口
    while (窗口不合法):
        把 s[l] 移出窗口
        l++
    更新答案
```

**本题中"窗口不合法"** = 出现重复字符。用 `Map<Character, Integer>` 记录每个字符出现次数即可。

### 3. 进一步优化：直接跳过

记录每个字符 **上次出现的下标** `last[c]`：若 `s[r]` 之前出现过且 `last[s[r]] >= l`，直接把 `l` 跳到 `last[s[r]] + 1`，省掉 inner while。

> **学习点 ①**：滑动窗口的"左端如何缩"有两种风格 —— **逐步缩** 通用，**直接跳** 更快但要写对边界。

### 4. 容易踩的坑

| 坑 | 处理 |
|---|---|
| 直接跳法忘判 `last[c] >= l`，错把窗口外的旧位置当数 | **必须**判 `>= l`，否则 `l` 会倒退 |
| 字符集没说定，开 `int[128]` vs `int[256]` 不确定 | ASCII 用 `int[128]` 够；扩展字符用 `HashMap` 最稳 |
| 空串边界 | `n=0` 直接返回 0 |

## 三、Java 题解

### 解法 A：滑动窗口 + 计数（通用骨架，推荐先掌握）

```java
class Solution {
    public int lengthOfLongestSubstring(String s) {
        int[] cnt = new int[128];
        int l = 0, best = 0;
        for (int r = 0; r < s.length(); r++) {
            char c = s.charAt(r);
            cnt[c]++;
            while (cnt[c] > 1) {        // 窗口含重复
                cnt[s.charAt(l)]--;
                l++;
            }
            best = Math.max(best, r - l + 1);
        }
        return best;
    }
}
```

**记忆口诀**：
> **"右扩入窗，左缩出窗，窗口合法再更新答案。"**

### 解法 B：哈希表直接跳（更快）

```java
class Solution {
    public int lengthOfLongestSubstring(String s) {
        int[] last = new int[128];
        Arrays.fill(last, -1);
        int l = 0, best = 0;
        for (int r = 0; r < s.length(); r++) {
            char c = s.charAt(r);
            if (last[c] >= l) l = last[c] + 1;   // 跳到上次出现之后
            last[c] = r;
            best = Math.max(best, r - l + 1);
        }
        return best;
    }
}
```

## 四、复杂度

| 解法 | 时间 | 空间 |
|---|---|---|
| 滑动窗口 / 直接跳 | **O(n)** | O(Σ)，Σ ≤ 128 |
| 暴力 | O(n³) | O(Σ) |

> 滑动窗口 O(n) 的关键：`l` 和 `r` 都最多走 n 步，**双指针不回头**。

## 五、示例验证

`s = "pwwkew"`

| r | c | 窗口 (l..r) | 操作 | best |
|---|---|---|---|---|
| 0 | p | [0..0] "p" | — | 1 |
| 1 | w | [0..1] "pw" | — | 2 |
| 2 | w | [0..2] → 出 'p'(l=1) → 出 'w'(l=2) | 窗口变 "w" | 2 |
| 3 | k | [2..3] "wk" | — | 2 |
| 4 | e | [2..4] "wke" | — | **3** |
| 5 | w | [2..5] → 出 'w'(l=3) | 窗口 "kew" | 3 |

输出 `3` ✅

## 六、复盘与延伸

### 一句话总结
> **滑动窗口三件套：右扩、左缩到合法、更新答案。**

### 新手常见疑问（FAQ）

**Q1：为什么右指针不需要“回头重算”？**
A：核心是单调性：固定左端 l 时，最优右端 r 随 l 增大也只会变大或不变，不会变小。所以 l 跳过某些位置后 r 从上次的位置继续向右即可。

**Q2：计数版为什么是 `while (cnt[c] > 1)` 而不是 `if`？**
A：本题实际上 `if` 也对 (因为只有 c 刚加1)，但 `while` 是滑窗通用模板，保持一致是好习惯。其他题（如 LC 76）需要循环缩。

**Q3：直接跳版 `last[c] >= l` 为什么要判 `>= l`？**
A：`last[c]` 记录的是全局上次出现位置，可能在当前窗口外（< l）。不判会让 `l` 倒退，破坏滑窗单调性。

**Q4：Unicode / 中文字符怎么办？**
A：`int[128]` 不够。改用 `HashMap<Character, Integer>`；或如果是补充字符面（emoji）需用 `int codePoint`，遍历也要改为 `s.codePoints()`。

**Q5：返回最长子串本身怎么改？**
A：更新 best 时记 `bestL = l`，最后返回 `s.substring(bestL, bestL + best)`。

### 面试官常见 follow-up
1. **"最多含 k 个不同字符的最长子串呢？"** → 同样模板，不合法条件改为 `不同字符数 > k`。即 **LC 340**。
2. **"恰好含 k 个不同字符呢？"** → 转化为「最多 k 个」减「最多 k-1 个」。即 **LC 992** 的经典套路。
3. **"允许替换至多 k 个字符后的最长重复字符子串？"** → 窗口内 max 计数 + (窗口长 - max) ≤ k。即 **LC 424**。
4. **"上求最长、下求最短在模板上什么区别？"** → 最长：不合法才缩，更新在 while 外；最短：合法就缩，更新在 while 内。
5. **"如果要返回所有最长不重复子串（并列）？"** → 扫一遍记录所有与 best 相等的 `[l, r]`，存起点集合。
6. **"数据流（文件只能顺读一次）呢？"** → 滑窗天然支持，因为 l 和 r 都是单向。
