# LeetCode 198. 打家劫舍 (House Robber)

> 难度：Medium　|　标签：动态规划　|	|	**DP 入门第二题 ⭐⭐⭐**

---

## 一、题目

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，**如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警**。

给定一个代表每个房屋存放金额的非负整数数组，计算你 **不触动警报装置** 的情况下，**一夜之内能够偷窃到的最高金额**。

**约束**

- `1 <= nums.length <= 100`
- `0 <= nums[i] <= 400`

**示例**

| 输入 | 输出 | 方案 |
|---|---|---|
| `[1,2,3,1]` | `4` | 偷 1 和 3：`1+3` |
| `[2,7,9,3,1]` | `12` | 偷 1,3,5：`2+9+1` |

---

## 二、解题思路（学习重点）

### 1. 状态定义

`dp[i]` = **考虑前 i+1 间房（下标 0..i），能偷到的最大金额**。

### 2. 状态转移：**偷 or 不偷 当前房**

对于第 `i` 间房，两种选择：
- **偷**：那么 `i-1` 不能偷 → `dp[i-2] + nums[i]`
- **不偷**：那么收益就是 `dp[i-1]`

取较大：
$$dp[i] = \max(dp[i-1],\; dp[i-2] + nums[i])$$

**初始条件**：
- `dp[0] = nums[0]`
- `dp[1] = max(nums[0], nums[1])`

> **学习点 ①**：**"选 vs 不选"** 是 DP 最常见的二选一状态转移。同模板：LC 213（环形）、LC 740（删除并获得点数）、LC 337（树形）。

### 3. 空间优化到 O(1)

`dp[i]` 只依赖 `dp[i-1]` 和 `dp[i-2]` → 两个变量滚动。

### 4. 容易踩的坑

| 坑 | 处理 |
|---|---|
| 误以为是"奇数下标 / 偶数下标"二选一 | 反例 `[2,1,1,2]` → 应偷下标 0 和 3，跳过 1 和 2 |
| 边界 `n=1` 或 `n=2` 没单独处理 | 用 `dp[0]=nums[0], dp[1]=max(nums[0],nums[1])` 统一 |

---

## 三、详细解题步骤（O(1) 空间版）

**步骤 1**：边界
```java
int n = nums.length;
if (n == 1) return nums[0];
```

**步骤 2**：初始化两个滚动变量
```java
int prev2 = nums[0];                          // dp[i-2]
int prev1 = Math.max(nums[0], nums[1]);       // dp[i-1]
```

**步骤 3**：从 `i = 2` 开始迭代
```java
for (int i = 2; i < n; i++) {
    int cur = Math.max(prev1, prev2 + nums[i]);
    prev2 = prev1;
    prev1 = cur;
}
return prev1;
```

---

## 四、Java 题解

### 解法 A：O(1) 空间（推荐）

```java
class Solution {
    public int rob(int[] nums) {
        int n = nums.length;
        if (n == 1) return nums[0];
        int prev2 = nums[0];
        int prev1 = Math.max(nums[0], nums[1]);
        for (int i = 2; i < n; i++) {
            int cur = Math.max(prev1, prev2 + nums[i]);
            prev2 = prev1;
            prev1 = cur;
        }
        return prev1;
    }
}
```

### 解法 B：DP 数组版（直观教学版）

```java
class Solution {
    public int rob(int[] nums) {
        int n = nums.length;
        if (n == 1) return nums[0];
        int[] dp = new int[n];
        dp[0] = nums[0];
        dp[1] = Math.max(nums[0], nums[1]);
        for (int i = 2; i < n; i++) {
            dp[i] = Math.max(dp[i-1], dp[i-2] + nums[i]);
        }
        return dp[n-1];
    }
}
```

**记忆口诀**：
> **"偷今天 + 前天，不偷就用昨天，取大者。"**

## 六、示例验证

`nums = [2, 7, 9, 3, 1]`

| i | prev2 | prev1 | nums[i] | cur = max(prev1, prev2+nums[i]) |
|---|---|---|---|---|
| init | 2 | 7 | — | — |
| 2 | 2 | 7 | 9 | max(7, 2+9)=11 |
| 3 | 7 | 11 | 3 | max(11, 7+3)=11 |
| 4 | 11 | 11 | 1 | max(11, 11+1)=**12** |

输出 `12` ✅

## 七、复盘与延伸

### 一句话总结
> **"偷 or 不偷" 二选一：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`。**

### 新手常见疑问（FAQ）

**Q1：能不能“偏偏偏，奇偏偶偏取较大者”？**
A：不能。反例 `[2,1,1,2]`：奇位 1+1=2，偶位 2+2=4。但最优是偊下标 0 和 3：2+2=4——刚好是偶位，这个例子看不出为什么不行。更强反例 `[2,1,1,5,1,2]`：偶位 2+1+1=4，奇位 1+5+2=8，但最优是 2+5+2=9（下标 0,3,5）。

**Q2：为什么不是 `dp[i] = dp[i-2] + nums[i]` 而要跟 `dp[i-1]` 比较？**
A：不偷 i 时，`dp[i] = dp[i-1]`，不一定是“偊 i-1”。定义记住：`dp[i]` 是“前 i+1 间能偊的最大金额”，不是“偊 i 间的最大”。

**Q3：prev1、prev2 更新顺序错了会怎样？**
A：必须 `prev2 = prev1; prev1 = cur;` 顺序。反了 prev2 就变成了 cur，下一轮算错。可以用中间变量 `tmp`。

**Q4：能否返回偊了哪些房？**
A：反向回溯。从 dp[n-1] 开始，若 `dp[i] != dp[i-1]` 则偊了 i，跳到 i-2；否则跳到 i-1。要返回具体下标需保留 dp 数组（无法用滚动变量）。

**Q5：面试官思路微调：“有三个相邻就报警”怎么改？**
A：`dp[i] = max(dp[i-1], dp[i-2] + nums[i], dp[i-3] + nums[i-1] + nums[i])`；报警限制 i-1, i-2, i-3 三者不能全偷。难度上升但模板不变。

### 面试官常见 follow-up
1. **"环形偊窃（首尾相邻）（LC 213）？"** → 分两次：偊 `[0, n-2]` 或 `[1, n-1]`，取较大。
2. **"树形偊窃（LC 337）？"** → DFS 返回 `int[]{不偷, 偷}`；在每个节点决定。
3. **"删除并获得点数（LC 740）？"** → 转化为偊窃：按值汇总到 `sum[x]`，则偊 x 后不能选 x±1。
4. **"返回偊了哪些房间？"** → 保留 dp 数组，反向回溯。
5. **"多个小偷合作（可同时偊不相邻的多间）？"** → 问题不变，还是 DP。本题本身就允许偊多间。
6. **"每间房被偊后要冷却 1 夜才能再偊（与状态机面试可全通）？"** → 状态机 DP，定义 `偊/不偊/冷却` 三状态。类 LC 309。
